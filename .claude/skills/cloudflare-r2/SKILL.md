# Cloudflare R2 Setup for Jumpstart Pro Rails

## Prerequisites

- `wrangler` CLI authenticated (`wrangler whoami` to get account ID)
- `cloudflare` npm package installed globally (`npm install -g cloudflare`)
- Auth via env: `CLOUDFLARE_API_KEY` + `CLOUDFLARE_EMAIL`
- Wrangler OAuth token cannot create API tokens (returns 403)

## Workflow

### 0. Ask Environment

Before running any command, ask the user which environment to target: `production` / `staging` / `development`. This determines:
- Which Rails credentials file to save to (`credentials/<env>.yml.enc`)
- Token type: production uses Account API Token, others use User API Token

### 1. Create R2 Bucket

```bash
wrangler r2 bucket create <bucket-name>
```

### 2. Create R2 Token and Save to Credentials

Run the bundled script (requires `cloudflare` npm package):

```bash
node <this-skill-path>/scripts/create-r2-token.mjs --bucket <bucket-name> --account <account-id> --env production [--cdn-host https://cdn.example.com]
```

The script will:
1. Create an R2 S3-compatible API token via Cloudflare API
2. Generate a random `cdn_secret` for HMAC-SHA256 signed URLs
3. Automatically save all credentials to Rails encrypted credentials (deep merge)

The `secret_access_key` is the SHA-256 hex digest of the token value returned by Cloudflare (see [R2 docs](https://developers.cloudflare.com/r2/api/tokens/)). The raw token value is only returned at creation time and cannot be retrieved again, so the script hashes it immediately and saves the hash — do not re-run with the same token name.

### 3. Configure CORS

Active Storage direct upload lets the browser PUT directly to the R2 endpoint — a cross-origin request. Without a CORS policy on the bucket, the browser's preflight blocks the upload and the file never reaches R2 (DB has a blob row, bucket is empty).

Run one CORS command per environment with the origins that env actually serves from:

```bash
# Example: dev bucket — local only
node <this-skill-path>/scripts/set-r2-cors.mjs \
  --bucket dev-example-com \
  --origins "http://localhost:3000,http://127.0.0.1:3000"

# Example: prod bucket — live domain only
node <this-skill-path>/scripts/set-r2-cors.mjs \
  --bucket example-com \
  --origins "https://example.com"
```

Don't pool all envs into one bucket's rule — keep dev origins out of prod CORS.

Defaults (override with `--methods` / `--max-age` if needed):
- `methods`: `GET, PUT, POST, HEAD`
- `headers`: `*`
- `exposeHeaders`: `ETag, Content-Length`
- `maxAgeSeconds`: `3600`

R2 does not support `*` wildcards in origins — list each port/host explicitly.

### 4. Verify Credentials

```bash
EDITOR=cat bin/rails credentials:edit -e production
```

Expected credential structure matching `config/storage.yml`:

```yaml
cloudflare:
  account_id: <account-id>
  r2:
    access_key_id: <auto-saved>
    secret_access_key: <auto-saved>
    bucket_name: <auto-saved>
    cdn_secret: <auto-generated>
    cdn_host: <from --cdn-host, if provided>
```
